# Commit Rule 

## 제목
- 첫 글자는 대문자로 입력합니다.
- 명령문으로 작성합니다.
- 끝에 마침표'.'를 작성하지 않습니다.
- 최대 50 글자까지만 입력합니다.
    - ex) <타입>: <제목>

## 본문
- 한 줄에 최대 72 글자까지만 입력합니다.
- 앞에 '-'를 작성합니다.
- 제목과 본문은 한 줄 띄워 분리합니다.
- "어떻게" 보다 "무엇을", "왜"를 설명합니다.

## <타입> 리스트
- feat    : 기능 (새로운 기능)
- fix     : 버그 (버그 수정)
- refactor: 리팩토링
- style   : 스타일 (코드 형식, 세미콜론 추가: 비즈니스 로직에 변경 없음)
- docs    : 문서 (문서 추가, 수정, 삭제)
- test    : 테스트 (테스트 코드 추가, 수정, 삭제: 비즈니스 로직에 변경 없음)
- chore   : 기타 변경사항 (빌드 스크립트 수정 등)
